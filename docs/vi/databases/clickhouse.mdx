# Kết nối ClickHouse

TablePro hỗ trợ ClickHouse thông qua giao diện HTTP. ClickHouse là cơ sở dữ liệu OLAP hướng cột được xây dựng cho phân tích thời gian thực trên tập dữ liệu lớn. TablePro kết nối qua HTTP (cổng 8123 mặc định), không phải giao thức TCP gốc.

## Cài đặt Plugin

Driver ClickHouse có sẵn dưới dạng plugin tải về. Khi bạn chọn ClickHouse trong form kết nối, TablePro sẽ tự động nhắc bạn cài đặt. Bạn cũng có thể cài đặt thủ công:

1. Mở **Cài đặt** > **Plugin** > **Duyệt**
2. Tìm **ClickHouse Driver** và nhấn **Cài đặt**
3. Plugin được tải và kích hoạt ngay lập tức, không cần khởi động lại

## Thiết lập Nhanh

<Steps>
  <Step title="Mở Biểu mẫu Kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn ClickHouse">
    Chọn **ClickHouse** từ bộ chọn loại cơ sở dữ liệu
  </Step>
  <Step title="Nhập Thông tin Kết nối">
    Điền host, port, username, password và tên database
  </Step>
  <Step title="Kiểm tra và Kết nối">
    Nhấp **Test Connection**, sau đó **Create**
  </Step>
</Steps>

## Cài đặt Kết nối

### Trường Bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Hostname hoặc địa chỉ IP máy chủ | `localhost` |
| **Port** | Cổng HTTP của ClickHouse | `8123` |

### Trường Tùy chọn

| Trường | Mô tả |
|-------|-------------|
| **Username** | Người dùng ClickHouse | `default` |
| **Password** | Mật khẩu người dùng (mặc định trống cho cài đặt cục bộ) |
| **Database** | Cơ sở dữ liệu để kết nối | `default` |

<Tip>
ClickHouse đi kèm với người dùng `default` không có mật khẩu trên bản cài đặt mới. Đối với máy chủ production, luôn đặt mật khẩu hoặc cấu hình kiểm soát truy cập.
</Tip>

## Giao diện HTTP

TablePro kết nối với ClickHouse thông qua HTTP API, không phải giao thức TCP gốc (cổng 9000). Giao diện HTTP được bật mặc định trên cổng 8123 và hỗ trợ tất cả các thao tác SQL.

Nếu máy chủ ClickHouse sử dụng HTTPS, hãy bật SSL/TLS trong biểu mẫu kết nối. Cổng HTTPS mặc định là 8443.

<Note>
Một số nhà cung cấp ClickHouse cloud chỉ cung cấp giao diện HTTP(S). TablePro hoạt động với các dịch vụ này ngay lập tức vì sử dụng HTTP nguyên bản.
</Note>

## Cấu hình Mẫu

### Máy chủ Phát triển Cục bộ

```
Name:     Local ClickHouse
Host:     localhost
Port:     8123
Username: default
Password: (trống)
Database: default
```

### Docker ClickHouse Container

Khởi chạy container ClickHouse để thử nghiệm cục bộ:

```bash
docker run -d \
  --name clickhouse \
  -p 8123:8123 \
  -p 9000:9000 \
  -e CLICKHOUSE_USER=default \
  -e CLICKHOUSE_PASSWORD=secret \
  clickhouse/clickhouse-server:latest
```

Sau đó kết nối với:

```
Name:     Docker ClickHouse
Host:     localhost
Port:     8123
Username: default
Password: secret
Database: default
```

### ClickHouse Cloud

```
Name:     ClickHouse Cloud
Host:     abc123.us-east-1.aws.clickhouse.cloud
Port:     8443
Username: default
Password: (mật khẩu cloud của bạn)
Database: default
```

<Note>
ClickHouse Cloud yêu cầu HTTPS (cổng 8443). Bật SSL/TLS trong biểu mẫu kết nối.
</Note>

### Máy chủ Từ xa

```
Name:     Production ClickHouse
Host:     clickhouse.example.com
Port:     8123
Username: app_user
Password: (mật khẩu bảo mật)
Database: analytics
```

<Warning>
Đối với các instance ClickHouse production, hãy cân nhắc sử dụng [SSH tunneling](/vi/databases/ssh-tunneling) nếu cổng HTTP không được mã hóa.
</Warning>

## Kết nối SSL/TLS

TablePro hỗ trợ mã hóa SSL/TLS cho kết nối ClickHouse. Cấu hình SSL trong phần **SSL/TLS** của biểu mẫu kết nối.

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL (HTTP trên cổng 8123) |
| **Required** | Yêu cầu mã hóa SSL (HTTPS, thường cổng 8443) |

<Tip>
ClickHouse Cloud và hầu hết các dịch vụ ClickHouse được quản lý yêu cầu HTTPS. Đặt cổng thành 8443 và bật SSL.
</Tip>

## Hỗ trợ SSH Tunnel

Bạn có thể kết nối với ClickHouse thông qua SSH tunnel để truy cập an toàn vào máy chủ từ xa. Xem [SSH Tunneling](/vi/databases/ssh-tunneling) để biết hướng dẫn thiết lập chi tiết.

## Tính năng

### Duyệt Cơ sở dữ liệu

Sau khi kết nối, sidebar liệt kê tất cả cơ sở dữ liệu trên máy chủ. Mở rộng một cơ sở dữ liệu để xem các bảng và view. Chuyển đổi cơ sở dữ liệu bằng bộ chuyển đổi trên thanh công cụ.

### Duyệt Bảng

Đối với mỗi bảng, TablePro hiển thị:

- **Structure**: Cột với kiểu dữ liệu ClickHouse, biểu thức mặc định và ghi chú
- **Indexes**: Chỉ mục bỏ qua dữ liệu (minmax, set, bloom_filter, v.v.)
- **DDL**: Câu lệnh CREATE TABLE đầy đủ bao gồm engine và cài đặt
- **Parts**: Chi tiết partition và part từ `system.parts` (số hàng, dung lượng đĩa, trạng thái active)

### Thông tin Engine

Bảng ClickHouse sử dụng các engine lưu trữ quyết định cách dữ liệu được lưu trữ và truy vấn. TablePro hiển thị loại engine và các tham số trong chế độ xem cấu trúc bảng. Các engine phổ biến:

| Engine | Trường hợp sử dụng |
|--------|----------|
| **MergeTree** | Engine chính cho bảng phân tích production |
| **ReplacingMergeTree** | Loại bỏ trùng lặp theo sorting key |
| **SummingMergeTree** | Tổng hợp trước các cột số |
| **AggregatingMergeTree** | Tổng hợp tăng dần với cột AggregateFunction |
| **Log / TinyLog** | Bảng nhỏ, phát triển và thử nghiệm |
| **Memory** | Bảng trong bộ nhớ cho dữ liệu tạm thời |
| **Distributed** | Truy vấn qua các shard trong cluster |

### Sorting Key và Partition Key

Bảng ClickHouse được sắp xếp theo sorting key (mệnh đề ORDER BY trong định nghĩa bảng). TablePro hiển thị sorting key, partition key và primary key trong chế độ xem cấu trúc bảng. Các key này rất quan trọng cho hiệu suất truy vấn vì ClickHouse đọc dữ liệu theo thứ tự đã sắp xếp.

### Chỉ mục Bỏ qua Dữ liệu

ClickHouse hỗ trợ chỉ mục phụ để bỏ qua các granule trong quá trình truy vấn. TablePro hiển thị chúng trong tab Indexes:

```sql
-- Ví dụ: chỉ mục bloom filter trên cột URL
ALTER TABLE hits ADD INDEX url_idx url TYPE bloom_filter GRANULARITY 4;
```

### Theo dõi Tiến trình Truy vấn

Trong quá trình thực thi truy vấn, thanh công cụ hiển thị tiến trình trực tiếp: số hàng đã đọc và bytes đã xử lý. Sau khi hoàn thành, hiển thị tóm tắt gồm thời gian thực thi, số hàng đã đọc và bytes đã đọc. Dữ liệu này được lấy từ `system.processes`.

### Biến thể EXPLAIN

ClickHouse hỗ trợ nhiều chế độ EXPLAIN. Nhấp vào menu thả xuống Explain trong trình soạn truy vấn để chọn:

| Biến thể | Mô tả |
|---------|-------------|
| **Plan** | Kế hoạch truy vấn logic (mặc định) |
| **Pipeline** | Pipeline thực thi vật lý với thông tin thread/port |
| **AST** | Cây cú pháp trừu tượng của truy vấn đã phân tích |
| **Syntax** | Truy vấn sau tối ưu hóa cú pháp |
| **Estimate** | Ước tính số hàng, marks và parts cần đọc |

### Hủy Truy vấn Phía Server

Khi bạn hủy truy vấn đang chạy, TablePro gửi lệnh `KILL QUERY` đến máy chủ ClickHouse. Điều này dừng truy vấn ở phía server, không chỉ kết nối HTTP.

### Trình soạn Truy vấn

Viết và thực thi truy vấn SQL ClickHouse trong trình soạn thảo:

```sql
-- Truy vấn tổng hợp
SELECT
    toDate(event_time) AS date,
    count() AS events,
    uniqExact(user_id) AS unique_users
FROM analytics.events
WHERE event_time >= now() - INTERVAL 7 DAY
GROUP BY date
ORDER BY date;

-- Bảng hệ thống
SELECT name, engine, total_rows, total_bytes
FROM system.tables
WHERE database = 'analytics';

-- Phân vùng bảng
SELECT partition, sum(rows) AS rows, formatReadableSize(sum(bytes_on_disk)) AS size
FROM system.parts
WHERE table = 'events' AND active
GROUP BY partition
ORDER BY partition;
```

### Chỉnh sửa Dữ liệu

TablePro hỗ trợ chỉnh sửa giá trị ô, chèn hàng và xóa hàng trong bảng ClickHouse. Các thay đổi được gửi dưới dạng câu lệnh INSERT, ALTER TABLE UPDATE và ALTER TABLE DELETE tiêu chuẩn.

<Warning>
Mutation ClickHouse (UPDATE và DELETE) là bất đồng bộ. Chúng thực thi ở chế độ nền và có thể mất thời gian để hoàn thành trên các bảng lớn. Xem phần Hạn chế để biết chi tiết.
</Warning>

### Xuất và Nhập

Xuất kết quả truy vấn hoặc dữ liệu bảng sang CSV, JSON và các định dạng khác. Nhập dữ liệu từ file CSV vào bảng ClickHouse.

## Cơ sở dữ liệu Hệ thống

ClickHouse bao gồm các cơ sở dữ liệu hệ thống tích hợp:

| Cơ sở dữ liệu | Mô tả |
|----------|-------------|
| **system** | Thống kê máy chủ, nhật ký truy vấn, thông tin parts, trạng thái cluster |
| **information_schema** | View metadata theo chuẩn SQL (tables, columns, schemata) |
| **INFORMATION_SCHEMA** | Alias cho `information_schema` (biến thể phân biệt chữ hoa/thường) |

Cơ sở dữ liệu `system` đặc biệt hữu ích cho giám sát:

```sql
-- Truy vấn gần đây
SELECT query, read_rows, elapsed, memory_usage
FROM system.query_log
WHERE type = 'QueryFinish'
ORDER BY event_time DESC
LIMIT 20;

-- Dung lượng đĩa theo bảng
SELECT database, table, formatReadableSize(sum(bytes_on_disk)) AS size
FROM system.parts
WHERE active
GROUP BY database, table
ORDER BY sum(bytes_on_disk) DESC;
```

## Xử lý Sự cố

### Kết nối bị Từ chối

**Triệu chứng**: "Connection refused" hoặc timeout

**Nguyên nhân và Giải pháp**:

1. **ClickHouse không chạy hoặc giao diện HTTP bị tắt**
   ```bash
   # Kiểm tra xem ClickHouse có đang lắng nghe trên cổng 8123 không
   curl http://localhost:8123/ping
   # Phản hồi mong đợi: "Ok."
   ```

2. **Sai cổng**
   - Giao diện HTTP: 8123 (mặc định)
   - Giao diện HTTPS: 8443
   - TCP gốc: 9000 (không được TablePro sử dụng)

3. **Giao diện HTTP bị tắt trong cấu hình**
   - Kiểm tra `<http_port>` trong `/etc/clickhouse-server/config.xml`

### Xác thực Thất bại

**Triệu chứng**: "Authentication failed" hoặc HTTP 403

**Giải pháp**:

1. Xác minh username và password
2. Kiểm tra người dùng tồn tại trong ClickHouse:
   ```sql
   SELECT name, auth_type FROM system.users;
   ```
3. Xác minh người dùng có quyền truy cập vào cơ sở dữ liệu đích:
   ```sql
   SHOW GRANTS FOR app_user;
   ```

### Timeout Kết nối

**Triệu chứng**: Kết nối bị treo hoặc timeout

**Giải pháp**:

1. Xác minh host và port chính xác
2. Kiểm tra kết nối mạng và quy tắc tường lửa
3. Đối với ClickHouse được lưu trữ trên cloud, đảm bảo IP của bạn nằm trong danh sách cho phép

## Hạn chế Đã biết

- **Không có foreign key.** ClickHouse không hỗ trợ ràng buộc foreign key. Tính toàn vẹn tham chiếu phải được quản lý ở tầng ứng dụng.
- **Không có transaction.** ClickHouse không hỗ trợ transaction nhiều câu lệnh. Mỗi câu lệnh thực thi độc lập.
- **Mutation bất đồng bộ.** Thao tác UPDATE và DELETE chạy dưới dạng mutation nền qua `ALTER TABLE UPDATE` và `ALTER TABLE DELETE`. Chúng không tức thì: trên bảng lớn, mutation có thể mất từ vài giây đến vài phút. Kiểm tra tiến trình mutation với:
  ```sql
  SELECT * FROM system.mutations WHERE is_done = 0;
  ```
- **Không có auto-increment.** ClickHouse không hỗ trợ cột tự tăng.
- **Primary key không thể thay đổi.** Primary key và sorting key không thể thay đổi sau khi tạo bảng.
- **Tối ưu cho INSERT.** ClickHouse được thiết kế cho chèn hàng loạt, không phải ghi từng hàng đơn lẻ. Chèn từng hàng một vẫn hoạt động nhưng không hiệu quả cho tải dữ liệu lớn.
